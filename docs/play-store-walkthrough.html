Title

Candela: Read Aloud

Short description

Free books, tech guides, and the web — read aloud by on-device voices.

Full description

Candela is TechEmpower's free, accessible resource app — read aloud. Browse free tech guides, find local help by dialing 211, reach a peer-support Discord in one tap, and read or listen to all of it through high-quality neural text-to-speech that runs entirely on your device. No subscription. No ads. No tracking. No account required. Under the hood, Candela is a serious audiobook player for any text you can reach — twenty-five fiction and reference sources are wired in. Browse Project Gutenberg, Standard Ebooks, LibriVox public-domain audiobooks, Royal Road, Archive of Our Own, Wikipedia, Wikisource, Hacker News, arXiv, PLOS, Google News, Bookshare's accessible library, and public libraries via the Palace Project. Open EPUB, PDF, ODT, or text files from your device, or turn a photo or scan into text with on-device OCR. Connect a Notion workspace, an Outline wiki, a Memory Palace you host yourself, or your Discord, Slack, Matrix, and Telegram channels. Tune in to internet radio with search across 30,000+ stations. Share any web link into Candela and a magic-link reader pulls the readable text and queues it for narration. Four on-device neural voice families ship, all fully offline once downloaded: Piper (45 English voices in US and UK accents), Kokoro (53 warm multi-speaker voices across nine languages, from Spanish and French to Hindi, Japanese, and Chinese), KittenTTS (the lightest tier, about 25 MB, for slower phones), and Supertonic (ten expressive, high-quality English voices). Your device's own voices work out of the box with zero download, and you can add a Microsoft Azure key for optional cloud voices — Candela takes no cut and falls back to a local voice if the network drops. Assign a different narrator to each book, and adjust speed, pitch, and pause length live. The hybrid reader shows the chapter in EB Garamond on a warm dark theme, highlighting the spoken sentence in brass as the voice glides along — or read along with an auto-scrolling teleprompter, look up any word with a tap, and search inside a book. Ask the built-in AI assistant about any book or get a chapter recap (bring your own key — Anthropic, OpenAI, or local Ollama). Listen on Android Auto, the lock screen, or a home-screen widget, with a sleep timer and gapless auto-advance. A Wear OS companion puts playback and a teleprompter remote on your wrist. Built for accessibility and for slower devices: TalkBack-aware navigation, WCAG AA brass-on-near-black contrast, OpenDyslexic and Atkinson Hyperlegible fonts, adjustable font scale, and reduced-motion options throughout. Optional cloud sync (powered by InstantDB) keeps your library, follows, reading positions, bookmarks, highlights and notes, pronunciation dictionary, and settings across devices — encrypted in transit, with any saved API keys end-to-end encrypted behind a passphrase only you hold. Delete synced data any time with the in-app "Delete cloud data" action. Or stay fully offline; your call. FEATURES • 25 fiction & reference sources — enable only the ones you want • 4 on-device neural voice families (Piper, Kokoro, KittenTTS, Supertonic) + your device's own voices • Optional bring-your-own-key cloud voices (Microsoft Azure) • Hybrid reader with synced brass highlighting + an auto-scrolling teleprompter • Tap-to-define dictionary, in-book search, and listening stats • Per-book AI chat & chapter recaps (bring your own key — Anthropic, OpenAI, Ollama) • Android Auto, lock-screen controls, sleep timer, home-screen widget • Make-your-own-audiobook (M4B) and EPUB export • Gapless chapter auto-advance with smart prerendering • Optional end-to-end-encrypted cross-device sync — or fully offline • Wear OS companion (playback + teleprompter remote) • Dark & light themes; OpenDyslexic & Atkinson fonts; TalkBack-friendly; WCAG AA contrast • Free, GPL-3.0, ad-free, no tracking, no account required Candela is built by TechEmpower, a 501(c)(3) nonprofit closing the digital divide with free tools and software. IV

Data Safety form

pre-drafted answers · enter at submission

The honest story: with sync off — the default — Candela collects nothing. A build-wide grep finds zero analytics, ads, or crash-reporting SDKs (the only third-party libraries are on-device ML Kit for OCR and the Compose fonts provider). Turn on optional sync and it stores your library and reading data on InstantDB; any saved API keys ride along end-to-end encrypted behind a passphrase only you hold.

VI

First release workflow

service account · credentials · first upload

The service account is what lets gradle upload without a browser. Set it up once, then the per-release flow is one command.

# local.properties (gitignored)
storyvox.playPublisher.credentialsFile=/home/jp/.storyvox-keystore/play-service-account.json

Then perform the first release:

# 1 · bump version in app/build.gradle.kts, update CHANGELOG.md
# 2 · tag + push (CI builds the AAB)
git tag -a v1.4.5 -m "v1.4.5 — six bug fixes + secrets audit"
git push origin main v1.4.5

# 3 · upload AAB → Internal Test as a draft, prints the edit URL
./gradlew :app:publishReleaseBundle

# 4 · open Play Console → Internal Test → click "Release"
If it says “requires credentials”Re-check storyvox.playPublisher.credentialsFile in local.properties and that the JSON exists.
VII

Per-release automation

gradle-play-publisher · one command, then promote

After the rites above, every future release runs through one command — Triple-T's gradle-play-publisher, configured in the play { } block of app/build.gradle.kts.

./gradlew :app:publishReleaseBundle   # → AAB to Internal Test, staged as DRAFT
SettingValueWhy
trackinternalAll uploads land in Internal Test; promote separately.
releaseStatusDRAFTStages as a draft; you click “Release”. CI can't auto-ship.
defaultToAppBundlestrueAAB is the canonical path; APK only on explicit ask.
commitfalseBuilds an “edit” without committing — prevents half-published state.
resolutionStrategyIGNOREOverrides a stale edit rather than failing; AGP versionCode is the source of truth.
# promote the tested internal build to production
./gradlew :app:promoteArtifact --from-track internal --promote-track production
Safety netEvery publishReleaseBundle is a draft until you click Release — a broken build never auto-publishes. The release keystore signs the AAB; the service-account JSON only has publishing rights (rotate immediately if exposed, via Cloud Console → Service Accounts → Keys).
VIII

Beta testing track

closed → open → production · testers before the world

Before going to Production, run the build through a Closed or Open testing track. Closed testing targets an invite list (up to 2,000 per group); Open testing is anyone with the link. Both give you crash reports, feedback, and a staged rollout before the public sees it.

Track vocabularyPlay Console calls them Internal, Closed, and Open testing. GPP uses the legacy names: internal, alpha (Closed), beta (Open). The promoteArtifact task accepts either naming style, but the track property in build.gradle.kts must use the legacy names.
TrackGPP nameAudienceReview?Best for
InternalinternalUp to 100 · email listNoCI drafts, smoke-test builds
ClosedalphaUp to 2,000 per groupYes (first time)Beta testers, org members, trusted users
OpenbetaAnyone with opt-in linkYes (first time)Public beta, wide feedback before GA
ProductionproductionEveryoneYes (first time)General availability

A · Create a Closed testing track

The fastest path: promote the already-tested Internal build to Closed, share the opt-in link with your testers.

No store listing neededClosed testing inherits your main store listing — you don't need separate screenshots or descriptions. The track is invisible to non-testers.

B · Promote a build to the beta track

Two paths: promote from Internal (recommended — the build is already tested) or upload directly.

# promote the current internal build → closed testing
./gradlew :app:promoteArtifact --from-track internal --promote-track alpha
# upload AAB directly to closed testing (override track)
./gradlew :app:publishReleaseBundle -Pplay.track=alpha
Review timelineThe first Closed testing release triggers a Google review (content rating, policy compliance). This typically takes a few hours to a couple of days. Subsequent releases to the same track usually clear within minutes.

C · Open testing (wider beta)

Optional step — skip if your Closed beta gives enough coverage. Open testing lets anyone opt in via a public link; no email list required.

# promote closed → open
./gradlew :app:promoteArtifact --from-track alpha --promote-track beta

# or skip closed, go internal → open
./gradlew :app:promoteArtifact --from-track internal --promote-track beta

D · Promote to Production

After the beta cycle is solid, promote the tested build to Production — same command from §VII, but now you have confidence from real tester feedback.

# promote from beta (open) → production
./gradlew :app:promoteArtifact --from-track beta --promote-track production

# or from closed → production (if you skipped open)
./gradlew :app:promoteArtifact --from-track alpha --promote-track production

# staged rollout: promote to 10% of production users first
./gradlew :app:promoteArtifact --from-track alpha --promote-track production --user-fraction 0.1
Staged rolloutPass --user-fraction 0.1 to roll out to 10% first. Monitor vitals for a day, then promote to 100% with --user-fraction 1.0. If something breaks, halt the rollout from Play Console → Production → Halt.

E · Typical release flow

Once the tracks are set up, a per-release beta cycle looks like this:

# 1 · tag + push → CI builds AAB
git tag -a v1.7.0 -m "v1.7.0 — new feature"
git push origin main v1.7.0

# 2 · upload to Internal Test (current default)
./gradlew :app:publishReleaseBundle

# 3 · smoke-test on Internal, then promote to Closed beta
./gradlew :app:promoteArtifact --from-track internal --promote-track alpha

# 4 · click "Release" in Play Console → Closed testing

# 5 · wait for tester feedback (hours/days) → fix if needed

# 6 · promote to Production (staged rollout optional)
./gradlew :app:promoteArtifact --from-track alpha --promote-track production
